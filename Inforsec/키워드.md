# 정보보호 관련 법령

1. 내부관리계획 - 정보통신서비스 제공자등이 개인정보의 안전한 처리를 위하여 개인정보보호 조직의 구성, 개인정보취급자의 교육, 개인정보보 보호조치 등을 규정한 계획

# 위험

1. 위험관리 - 조직자산에 대한 위험을 수용할 수 있는 수준으로 유지하기 위해 자산에 대한 위험을 분석하여 위험으로부터 자산을 보호하기 위해 비용대비 효과적인 보호대책을 마련하는 일련의 과정

2. 위험관리 필요성 

- 적절한 보호 대책을 우선순위에 따라 효율적으로 필요한 곳에 설치하도록 함으로써 과도하거나 과소의 투자를 예방

- 보호대책은 시간이 지나면서 그 효과가 감소하게 되므로, 조직의 위험수준을 지속적으로 유지하기 위해서는 지속적이고 반복적인 위험관리가 필요

3. 위험도 - 위험의 규모(위험의 발생가능성과 그 영향도의 조합)

4. 잔존위험 - 위험 조치 후에 남아있는 위험

5. 위험 분석 - 위험도를 결정하기 위해 위험을 파악하는 절차

6. 위험 평가 - 위험 식별과 위험 분석, 위험수준 평가를 포괄하는 절차

7. 위험 기준 - 평가된 위험의 경중을 판단하는 기준

8. 위험수준 평가 - 위험을 수용할지 조치할지 결정하기 위하여 위험분석 결과와 위험기준을 비교하는 절차

9. 위험 식별 - 위험을 식별하는 절차

10. 위험 관리 - 조직이 위험을 통제하기 위해 하는 활동

# 리눅스

1. tcpdump - 네트워크에서 송수신 되는 패킷을 스니핑(캡처) 할 수 있는 tool

- tcpdump -i {인터페이스명} (-n tcp or udp) (src or dst) host {IP주소} and (src or dst) port {Port 번호}

2. snort - 실시간 트래픽분석과 IP네트워크에서의 패킷 처리가 가능한 오픈소스 IDS SW

- alert tcp {소스(출발지) IP 주소 or any} {Port 번호 or any} -> {목적지 IP 주소 or any} {Port 번호 or any} (msg:"SSH Brute force login"; content:"SSH-2.0"; nocase; threshold:type both, track by_src,count 3,seconds 30;sid:1000110;)

3. iptables - 리눅스 커널 레벨에서 방화벽 기능을 제공해 주는 응용 프로그램

- iptables {Table}(default = filter, nat, mangle) {Chain}(input, output, forward) {Rule}(-p tcp(프로토콜), -s ip(출발지 ip), -d ip(목적지 ip), --sport(출발지 port), --dport(목적지 port)) {Target}(-j Accept, Drop, Reject, Log)

# 정보 시스템 공통 평가 기준

1. 보호 프로파일(PP) - 평가 대상 범주를 위한 특정 소비자의 요구에 부합하는 구현에 독립적인 보안 요구사항의 집합을 말한다.

2. 보안 목표명세서(ST) - 식별된 평가 대상의 평가를 위한 근거로 사용되는 보안 요구사항과 구현 명세의 집합

3. 평가 보증등급(EAL) - 공통 평가 기준에서 미리 정의된 보증 수준을 가지는 보증 컴포넌트로 이루어진 패키지를 말한다.

# 정보보안

1. 개인정보처리자 - 업무를 목적으로 개인정보파일을 운용하기 위하여 스스로 또는 다른 사람을 통하여 개인정보를 처리하는 공공기관, 법인, 단체 및 개인 등을 말한다.

2. 개인정보취급자 - 개인정보처리자의 지휘, 감독을 받아 개인정보를 처리하는 임직원, 파견 근로자, 시간제 근로자 등을 말한다.

3. 개인정보처리시스템 - 개인정보를 처리할 수 있도록 체계적으로 구성한 데이터베이스 시스템을 말한다.

4. 정보주체 - 처리되는 정보에 의하여 알아볼 수 있는 사람으로서 그 정보의 주체가 되는 사람

5. 개인정보파일 - 개인정보를 쉽게 검색할 수 있도록 일정한 규칙에 따라 체계적으로 배열하거나 구성한 개인정보의 집합물

6. 개인정보 보호책임자 - 개인정보처리자의 개인정보 처리에 관한 업무를 총괄해서 책임지는 자

7. 위험도 평가(분석) - 개인정보 유출에 영향을 미칠 수 있는 다양한 위험요소를 식별/평가하고 해당 위험요소를 적절하게 통제할 수 있는 방안 마련을 위해 종합적으로 분석하는 행위

8. 바이오정보 - 지문, 얼굴, 홍채, 정맥, 음성, 필적 등 개인을 식별할 수 있는 신체적 또는 행동적 특징에 관한 정보로서 그로부터 가공되거나 생성된 정보를 포함한다.

9. 내부망 - 물리적 망분리, 접근통제시스템 등에 의해 인터넷 구간에서의 접근이 통제 또는 차단되는 구간을 말한다.

10. 내부관리계획 - 개인정보처리자가 개인정보를 안전하게 처리하기 위하여 내부 의사결정 절차를 통하여 수립/시행하는 내부 기준

11. 정보공유 분석센터(Information Sharing & Analysis Center) - 주요정보통신기반시설에 대한 사이버 침해에 대해 관련 분야별 공동 대응을 하기 위해 만들어진 관리체계로,

- 취약점 및 침해요인과 그 대응방안에 관한 정보를 제공하고, 침해사고가 발생하는 경우 실시간 경보 및 분석체계를 운영하는 것을 목적으로 한다.

# 침해사고 분석 및 대응

1. 침입 탐지 시스템(IDS) - 네트워크나 시스템에 침입하는 악의적인 공격 등의 행위를 탐지하는 정보보호시스템

2. 침입 방지 시스템(IPS) - 네트워크나 시스템에 대한 다양한 침입 행위를 실시간 탐지, 분석하여 공격 패킷인 경우 자동으로 차단하는 시스템

3. 침입 차단 시스템(Firewall) - 네트워크 구간에 전송되는 패킷들을 정해진 규칙에 따라 차단하거나 통과시켜 공격자로부터 내부 자산을 보호하는 시스템

4. 가상 사설망(VPN) - 인터넷과 같은 공중망을 마치 사설망을 이용하듯 네트워크를 안전하게 연결하기 위한 통신 터널을 만들어 데이터를 안전하게 전송하는 시스템

5. 오용 탐지, 지식 기반 탐지, 시그니처 기반 탐지 - 특정 공격에 관한 분석 결과를 바탕으로 패턴을 설정 (미탐이 높고 오탐이 낮다)

6. 이상 탐지, 행위 기반 탐지 - 정상 행위를 벗어나는 데이터를 탐지해내는 기법 (미탐이 낮고 오탐이 높다)

7. 미탐(False Negative) - 실제 공격을 탐지하지 못하는 경우

8. 오탐(False Positive) - 공격이 아닌 것을 공격으로 탐지하는 경우

9. ModSecurity - Trustwave 사에 의해 제공되며 아파치 웹 서버, IIS 웹 서버 등을 지원하는 오픈소스 웹 방화벽

- 대표적인 공개 웹 방화벽에는 KISA의 캐슬(CASTLE), ATRONIX 사의 Webknight, Trustwave 사의 ModSecurity가 있다.

10. 통합 보안 시스템(UTM) - 방화벽, 침입 탐지/방지 시스템, 가상 사설망, 안티 바이러스 등 다양한 보안 기능을 하나의 장비로 통합적으로 제공하고 관리하는 보안 장비 <- 장비!

11. ESM(Enterprise Security Management) - 방화벽, 침입 탐지/방지 시스템 등 각종 보안 시스템의 로그들을 모아 한 곳에서 통합 관리를 할 수 있게 해주는 시스템 (로그 관리)

```

ESM 에이전트 : 관리 대상 보안 장비에 설치되어 로그 및 이벤트 데이터를 수집하여 ESM 매니저로 전달하는 기능을 수행

ESM 매니저 : 에이전트를 통해 수집된 데이터를 저장, 분석하여 그 결과를 콘솔로 전달하는 기능을 수행

ESM 콘솔 : 매니저에 의해 전달된 정보의 시각적 전달, 상황판단 및 리포팅 기능을 제공하며 ESM 매니저와 에이전트에 대한 제어/통제를 수행한다.

```

12. SIEM(Security Information & Enterprise Management) - ESM의 진화된 형태로 로그/이벤트 수집 및 관리를 위해 다음과 같은 기능을 제공한다.

```

로그 수집 : 관제 대상 시스템에 설치된 에이전트로 SNMP, syslog 서버에 저장하는 과정

로그 분류 : 이벤트 발생 누적 횟수 등 유사 정보를 기준으로 그룹핑하여 한 개의 정보로 취합하는 과정

로그 변환 : 다양한 로그 표현 방식을 표준 포맷으로 변환하는 과정

로그 분석 : 표준 포맷으로 변환된 로그 중에서 타임스탬프, IP주소, 이벤트 구성된 규칙을 기준으로 여러 개의 로그들의 연관성을 분석하는 과정

```

13. 보안관제 및 대응 절차

- IDS 및 IPS 등 보안 장비를 이용하여 실시간 트래픽 현황과 공격 탐지로그의 추이를 파악

- 최초 이상 징후가 발생되면 관제 담당자에게 보고하고 각각의 장비별 공격 탐지지표 현황을 비교, 분석하여 공격 탐지 여부를 결정

- 해당 정보보호시스템 및 장비를 이용하여 공격에 대한 초동 조치를 실시

- 장비별 공격 탐지표에 해당되는 정보를 수집하여 장비 간 상호 연관성 및 피해내용 등을 고려하여 공격 유형을 식별하고 피해 범위를 분석

- 해당 공격에 대하여 공격 유형 및 대응 매뉴얼에 따라 차단을 적용

14. 국가 사이버안전센터에서는 "국가 사이버안전관리규정"에 의거 사이버 공격에 체계적으로 대응하기 위해 파급 영향, 피해규모 등을 고려하여 수준에 따라 정상->관심->주의->경계->심각 단계로 사이버 위기 경보를 발령한다.

15. 퍼저(Fuzzer) - 어플리케이션의 보안 취약점을 찾아내기 위한 도구로 목표 어플리케이션에 대한 다양한(비 정상적인) 데이터를 입력함으로써 소프트웨어 오류를 유발하여 보안 취약점을 찾아내는 목적으로 사용하는 것

16. Windows PE(Portable Executable) 파일의 섹션 정보

```

.text : 프로그램 실행 코드를 담고 있는 섹션

.data : 읽기/쓰기가 가능한 영역으로 전역변수와 정적변수 등이 위치하는 섹션

.rdata : 읽기 전용 데이터 섹션

.bss : 초기화 되지 않은 전역변수가 위치한다.

.idata : 임포트 할 DLL과 그 API함수들에 대한 정보

.edata : 익스포트 할 DLL과 그 API함수들에 대한 정보

.rsrc : 다이얼로그, 아이콘, 커서 등의 윈도우 어플리케이션 리소스 관련 데이터를 담고 있는 섹션


```

17. Windows 레지스트리 루트 키

```

HKCR : HKEY_CLASS_ROOT - 파일 확장명과 응용 프로그램의 연결 정보가 들어있다.

HKCU : HKEY_CURRENT_USER - 현재 로그인 중인 사용자의 환경설정/프로파일 정보를 가지고 있다.

HKLM : HKEY_LOCAL_MACHINE - 시스템 전체에 적용되는 하드웨어(드라이버, 프린터 등)와 응용 프로그램의 설정 데이터를 저장한다.

HKU : HKEY_USERS - 각 사용자별로 존재하는 하이브 파일인 ntuser.dat 파일을 로드하여 생성된다. 다중 사용자 환경에서 각 사용자별로 키 항목을 생성하여 환경설정/프로파일 정보를 저장한다.

HKCC : HKEY_CURRENT_CONFIG - 윈도우의 하드웨어 프로필 정보(디스플레이, 폰트정보 등)를 가지고 있다.

```

18. 정적 프로그램 분석 (Static Program Analysis) - 실제 실행 없이 컴퓨터 소프트웨어를 분석하는 것 

- 대부분의 경우에 분석은 소스 코드의 버전 중 하나의 형태로 수행되며, 가끔은 목적 파일 형태로 분석된다.

- 프로그램의 이해를 통해 인간에 의한 자동화된 툴을 사용한 분석, 또는 코드 검토에 적용된다.

19. 동적 프로그램 분석 (Dynamic Program Analysis) - 실제 또는 가상 프로세서에서 프로그램을 실행함으로써 수행된다.

- 이것을 좀 더 혀과적으로 하기 위해서 프로그램은 반드시 충분히 테스트된 입력을 사용해서 흥미로운 행동을 생산해야 한다.

- 유닛 테스트, 통합 시험, 시스템 검사 그리고 인수 검사는 동적 분석을 사용한다.

20. 블랙박스 검사(Black-Box Testing) - 소프트웨어 검사 방법 중 하나로 어떤 소프트웨어를 내부 구조나 작동 원리를 모르는 상태에서 소프트웨어의 동작을 검사하는 방법을 이르는 말이다. <- 동작

- 유닛 검사, 인테그레이션 검사, 기능 검사, 시스템 검사, 적합성 검사

21. 화이트박스 검사(White-Box Testing) - 응용 프로그램의 내부 구조와 동작을 검사하는 소프트웨어 테스트 방식으로, 블랙박스 검사와는 반대된다.

- 제어 흐름 테스트, 데이터 흐름 테스트, 분기 테스트, 경로 테스트

22. 공급망 공격 - S/W 개발사의 네트워크에 침투하여 소스코드를 수정하여 악의적인 목적의 코드를 삽입하거나 배포를 위한 서버에 접근하여 악의적인 파일로 변경하는 방식을 통해 사용자 PC에서 소프트웨어 설치 또는 업데이트 시에 자동으로 감염되도록 하는 공격

23. APT(Advanced Persistent Threat) - 일반적으로 특정한 목표 대상에 대해 취약점을 파악하고 다양한 방법을 이용한 지속적인 공격 활동으로 정보 탈취, 

- 시스템 파괴 등의 손상을 입히려는 새로운 공격 형태로 비교적 공격 기법이 정교하여 탐지 및 차단이 어려운 공격 기법

24. SSDP DRDoS 공격 - Reflection 공격의 일종으로 UDP 1900 번 포트를 사용하여 IoT 시스템을 공격하는 기법

- SSDP는 홈 네트워크 미들웨어 표준인 UPnP에서 사용하는 프로토콜

25. POODLE 공격 -> TLS 연결을 3.0으로 다운그레이드하여 SSL 3.0 취약점을 이용하여 암호문을 해독하는 공격기법

26. 바이러스 -> 정상적인 파일이나 시스템 영역에 침입하여 그곳에 자신의 코드를 삽입하거나 설치하는 프로그램을 말한다.

27. 마이너 악성코드 (Miner Malware) - 사용자 모르게 PC에 상주하면서 PC의 자원을 암호 화폐를 채굴하는 데 사용한 후, 얻어진 암호 화폐를 공격자에게 전송하는 채굴 악성코드

28. 크립토 재킹(Crypto Jacking) - 피해자가 가지고 있는 암호 화폐를 탈취하는 행위

29. 워너크라이(Wanna Cry) - 윈도우 OS의 SMB(Server Message Block) 취약점을 이용한 공격, 취약한 포트 - TCP 139/445, UDP 137/138

30. 페티야(Petya) - Office 파일에 랜섬웨어 코드를 삽입하고 피해자에게 메일을 발송, 하드 디스크의 MFT(Master File Table)를 암호화한다. MBR(Master Boot Record)을 파괴하고, 공격자는 비트코인을 요구한다.

31. Nimda virus - 바이러스에 감염된 이메일의 첨부파일을 실행하지 않고, 본문 내용을 보기만 해도 자동으로 감염된다.

32. 코드 레드 - 웜 바이러스의 일종으로 서버에 숨어있다가 정해진 시간에 공격대상 웹 사이트로 대량의 쓰레기 정보를 보내 해당 웹 사이트를 마비시키는 악성 웜 바이러스

33. 아나토바(Anatova) 랜섬웨어 - 게임이나 앱의 아이콘으로 위장, 메모리를 비워 랜섬웨어 내용을 삭제하고, 백업 파일을 10회 이상 덮어쓰기하여 사용자가 복호화 키 없이 복구할 수 없도록 한다.

34. 폼 재킹(Form Jacking) - 온라인 구매 사이트에 악성 코드를 삽입하여 쇼핑객의 "카드 결제정보"를 탈취하는 공격기법

35. 딥 페이크(Deep Fake) - 딥러닝과 가짜의 결합어인 딥페이크는 인종지능 기술을 기반으로한 가짜 이미지, 오디오, 비디오를 뜻한다.

36. 멤캐시드(Memcached) - 멤캐시드 서버를 이용한 DDos 공격으로, 기존 DDoS와는 달리 위장된 명령어를 멤 캐시드 서버에 보내 공격을 반사 및 증폭시켜 막대한 양의 트래픽을 발생시키는 매우 위협적인 신종 공격 

37. 워터링 홀 공격 - 공격 대상이 방문할 가능성이 있는 합법적인 웹 사이트를 미리 감염시킨 뒤, 잠복하고 있다가 공격대상이 방문하면 대상의 컴퓨터에 악성코드를 설치하는 방법으로 공격을 진행

38. 하지메 봇넷 - 특정 유형의 기기만 공격하지 않고 인터넷의 모든 기기를 공격하는데, 활동이 집중되는 유형은 있다. 대부분은 디지털 비디오 레코더(DVR), 웹 카메라, 라우터이다.

39. 페르시라이 봇넷 -  IP 카메라 모델을 대상으로 함. TCP 81번 포트를 통해서 기기 검색을 한다. 제로데이 취약점 이용 (공격자가 인증 과정을 우회하여 관리자 권한으로 접속, 원격코드 실행, 공격자가 또 다른 Malware를 기기에 설치한다)

40. 다운로더 - 악성코드에서 지정한 외부 주소에 접속하여 추가 악성코드를 다운로드하여 실행시키는 악성코드

41. 드롭퍼 - 다운로드와 유사하게 새로운 악성코드를 생성하지만, 자신 내부에 포함되어 있는 데이터를 이용하여 악성코드를 생성

42. 인젝터 - 자신의 데이터를 이용해 바로 새로운 프로세스를 생성하여 메모리에 상주시키는 형태

43. 랜섬웨어 - 사용자 파일을 암호화 시켜 일정 시간안에 일정 금액을 지불하면 암호를 풀어주는 방식

44. 애드웨어 - 상업용 광고 목적으로 만들어진 악성코드로, 웹 브라우저 시작페이지 변경

45. 스파이웨어 - 사용자 PC에 사전 동의 없이 설치되어 컴퓨터 정보와 개인정보를 수집하는 악성코드

46. 트로이목마 - 정상적인 프로그램을 가장한 정보 유출, 자기복제 능력이 없다

47. 논리폭탄 - 특정 조건이 발생할 때 실행되는 악성코드

48. 키로거 - 사용자가 입력하는 정보를 갈취하는 악성코드

49. 백도어 - 인증과정을 우회해서 시스템에 접근할 수 있도록 열어둔 뒷문

50. 루트킷 - 악의적인 행동을 하는 프로그램을 숨기기 위한 목적으로 시작된 악성코드

51. 부트킷 - PC의 부팅영역인 MBR(Master Boot Record)을 조작하는 프로그램

52. APT - 경제적/사회적 목적을 위하여 다양한 IT 기술과 방식, 보안 위협을 이용하여 특정 대상을 지속적으로 공격하는 특징

53. 나이트 드래곤 - 2009년 1년 이상 카자흐스탄, 그리스, 대만과 미국에 위치한 글로벌 오일, 가스 및 석유 화학업체를 대상으로 조직적으로 진행되었다.

54. 스턱스넷 - 2010년 7월, 이란 원자력 발전소 작동을 방해한 악성코드로 SCADA 시스템을 임의로 제어하는데 사용되었다.

55. 웹쉘 - 원격에서 웹 서버에 명령을 수행할 수 있도록 작성한 웹 스크립트(ASP, JSP, PHP, CGI) 형태를 가진다.

56. 좀비프로세스 - 실행은 종료되었지만 아직 삭제되지 않은 프로세스. ps 명령어 실행 후 STAT 값이 'Z'로 표시되는 프로세스가 좀비 프로세스, Kill 명령어를 사용한다.

57. rlogin - 원격 시스템에서 접속할 때 사용하는 서비스, 사전에 서버의 /etc/hosts.equiv 파일에 호스트를 등록한 후 클라이언트는 패스워드를 입력할 필요없이 로그인이 가능하도록 하는 서비스

58. 리버스 텔넷 - 내부망에서 외부망으로 Telnet을 통해 연결하는것, 즉 내부망에서 공격자의 PC로 연결을 요청하는 것

59. 수리카타 - 오픈소스 IDS/IPS로서 기존의 스노트(Snort)의 장점을 수용하고, 대용량 트래픽을 실시간으로 처리하는데 특화된 소프트웨어

60. PAM(Pluggable Authentication Modules) - 공유 라이브러리를 사용하여 응용 프로그램이 어떤 방법으로 사용자를 인증할 것인지를 설정한다.

- 인증 모듈로, 사용자에 대한 인증을 각 응용 프로그램에서 하는 것이 아니라, 시스템 자체 내에서 계정을 생성하고 인증하도록 하는 메커니즘

- 접근 통제와 파일 시스템 보호 방식으로 Login, FTP, Telnet 등의 일반적인 서비스 인증 부분에 접근 통제를 설정한다.

61. 포트 미러링(Port Mirroring) - 스위치에 흐르는 데이터를 분석하기 위해 사용하는 기술이며, 해당 포트에 흐르는 트래픽을 트래픽 분석 장비가 설치된 포트로 복사하여 보내는 기술

62. NAC(Network Access Control) - 단말기가 네트워크에 접속을 시도할 때, 이를 제어하고 통제하는 기능을 제공하는 네트워크 보안 솔루션

63. John The Ripper - 다양한 패스워드 형식을 가지는 파일을 점검하는 도구, 공격자가 사용하는 패스워드 크래커로써 많이 사용되는 도구

64. Portsentry - 실시간으로 포트 스캔을 탐지하고 대응하기 위해 만들어진 프로그램

65. Tripwire - 대표적인 무결성 점검도구

66. nmap - 네트워크 보안을 위한 유틸리티로, 대규모 네트워크를 고속으로 스캔하는 도구

67. 시큐어코딩 - 소프트웨어의 개발 과정에서 개발자의 지식 부족이나 실수, 또는 각 프로그래밍 언어의 고유한 약점 등 다양한 원인으로 발생할 수 있는 취약점을 최소화하기 위하여 설계 단계부터 보안을 고려하여 코드를 작성하는 제작방식을 의미한다.

68. Web DAV - HTTP 사양에 대해 확장판으로서 분산형 제작 및 버전처리. 인가된 사용자에게 원격으로 웹 서버상에서 콘텐츠를 추가하고 관리할 수 있는 기술

69. UTM(Unified Treat Management) - 하나의 애플리케이션에서 장비에 여러가지 보안기능의 솔루션을 탑재하여 대응하고 관리할 수 있는 솔루션

- 다양한 보안솔루션(IDS, IPS, Firewall, VPN 등) 기능을 하나로 통합하여 보안 문제를 쉽고 편리하게 관리 및 해결하는 통합 보안관리 시스템

70. SAINT - 네트워크 취약점 분석도구. HTML 형식의 보고서 기능. 원격 취약점 점검 기능

71. SARA - SATAN을 기반으로 개발된 취약점 분석도구. 네트워크 기반의 컴퓨터, 서버, 라우터, IDS에 대한 취약점 분석이 가능하며, HTML형식의 보고서 기능

72. Nessus - 네트워크 취약점 점검 도구. 클라이언트와 서버 구조로 클라이언트의 취약점을 점검한다. 약 600여개 이상의 보안 취약점 점검 가능

73. Active Jamming - RFID 태그와 판독기 사이에 사전에 약속된 형태의 무의미한 전자신호를 지속적으로 발생시켜 불법적인 판독기가 태그와 통신하지 못하게 함으로써 RFID 태그의 프라이버시를 보호하는 방법

74. Kill Tag - 사용자가 제품을 구매하는 순간에 판독기가 Kill 명령어를 통하여 제품에 부착된 태그가 더 이상 동작하지 않도록 하는 방법

75. Shiled The Tag - RFID 태그에 사용되는 특정 주파수가 통과할 수 없도록 패러데이 케이즈 등의 전파 차단막으로 태그를 감추어 임의의 판독기가 태그에 접근하는 방법을 막는 것

76. Bit Locker - 윈도우 운영체제의 보안 기술에 포함된 완전한 디스크 암호화 기능. 볼륨 전체에 암호화 제공

77. Secure Copy - 네트워크 상에서 안전하게 파일을 복사할 수 있도록 해주는 유틸리티로, 데이터 전송과 사용자 인증을 위하여 SSH를 사용한다.

78. SOAR(Security Orchestration, Automation and Response) - 보안 오케스트레이션. 자동화 및 대응, 보안 운영 시 유입되는 다양한 보안 위협에 대해 대응 수준을 자동으로 분류

78. Threat Hunting - 자동화 된 방어 및 탐지 (Preventative & Detective) 솔루션에서 파악하기 어려운, 시스템에 숨겨진 고도화 된 위협을 발견하기 위한 중심의 프로세스를 뜻한다.

- 현재 Threat Hunting에서 가장 효과적인 가시성 확보 도구로 각광받고 있는 것이 "EDR"이다.

80. EDR(Endpoint Detection & Response) - 엔드포인트 레벨에서 지속적인 모니터링과 대응을 제공하는 보안 솔루션으로,

- 보안 사고탐지, 엔드포인트에서의 보안 사고 통제, 보안 사고조사, 감염전 상태로 엔드포인트 치료 등 4가지 기능을 제공한다.

81. 심층 패킷 검사 (Deep Packet Inspection, DPI) - 기본적으로 패킷 내부의 콘텐츠까지 파악하고, 클라이언트와 서버간의 패킷 통신의 규약에 대한 정보까지 파악하여 규약대로 통신이 이루어지고 있는지,

- 아니면 비 정상적인 통신 형태가 이루어지고 있는지를 파악하는 차세대 방화벽 기술

- 네트워크 트래픽을 조정하기 위해 쓰이는 기술

82. 이상 금융거래 탐지시스템 (Fraud Detection System, FDS) - 고객 신용카드 거래를 분석하여 평소와 다른 의심거래 이상 징후가 있을 때, 회원과 통화하거나 확인 과정을 통해 사고를 예방하는 시스템

83. 바이너리 디핑 (Binary Diffing) - APT 공격이 아닌, 제로데이 공격 취약점을 찾을 수 있는 기법. 디핑 기술을 사용해서 스크립트 된 바이너리 함수 정보를 획득한다.

- 즉, 자동으로 Malware를 탐지하고 오픈소스 라이센스 준수 여부를 확인할 수 있다.

# 어플리케이션보안

1. Recursive DNS 서버 - 클라이언트의 DNS 질의에 대한 응답 결과를 캐시에 일정시간 저장하고 있다가 동일한 DNS 질의가 들어오면 저장해 둔 정보를 이용하여 즉시 응답해주는 DNS 서버

2. DNS 서버 설정파일 - /etc/named.conf

3. DNS Cache Poisoning - 취약한 DNS 서버에 조작된 쿼리를 전송하여 DNS 서버가 가지고 있는 주소 정보를 임시적으로 변조하는 공격

4. nslookup - DNS 서버로부터 여러가지 정보를 얻을 수 있는 명령어. 호스트 이름으로부터 IP를 얻어내고 네임 서버를 운영하면서 문제가 생겼을 때 트러블 슈팅 시 많이 쓰이게 된다.

5. SNMP(Simple Network Management Protocol) - TCP/IP 네트워크 장치로부터 여러 관리 정보를 수집하고 실시간으로 상태를 모니터링 및 설정할 수 있는 인터넷 표준 프로토콜

- 매니저가 에이전트에게 정보를 요청하면 응답해주는 Manager -> Agent = 161/UDP, Polling

- 이벤트 발생 시 Agent -> Manager = 162/UDP, Trap

6. syslog - 실시간 전송방식과 유사하게 장비에서 발생되는 실시간 이벤트 정보를 서버에 전송하는 방식 <- 실시간

7. DHCP(Dynamic Host Configration Protocol) - 케이블 모뎀 사용자들이 많이 사용하고 있는 동적 IP 주소 할당 서비스

8. SMTP(Simple Mail Transport Protocol) - TCP/IP의 응용계층 프로토콜의 하나로 컴퓨터 간에 전자우편을 전송하기 위한 프로토콜

9. PGP(Pretty Good Privacy) - 컴퓨터 파일을 암호화하고 복호화하는 프로그램. 전 세계적으로 이메일 보안의 표준, 인터넷에서 사용되는 전자 우편 보안 도구로서, 전송되는 내용을 암호화하여 전송하는 기법

10. PEM(Privacy Enhanced Mail) - 응용계층에서 구현되는 전자 우편 도구로서 하위 프로토콜, 운영체제, 호스트와는 독립적으로 동작하고, 기존의 전자 우편 시스템에 도입하여도 별도의 변경 없이 동작할 수 있다.

11. S/MIME(Secure/Multipurpose Internet Mail Extensions) - 응용 계층에서 보안을 제공하는 가장 대표적인 시스템으로 MIME 객체에 암호화와 전자서명 기능을 추가한 프로토콜을 말한다.

- S/MIME은 기존 전자우편 보안 시스템의 문제점인 PEM 구현의 복잡성, PGP의 낮은 보안성과 기존 시스템과의 통합이 용이하지 않다는 점을 보완하기 위해 IETF의 작업 그룹에서

- RSADSI(RSA Data Security Incorporation)의 기술을 기반으로 개발된 전자우편 보안 시스템이다.

- S/MIME이 달성하고자 하는 목표는 강력한 암호화, 디지털 서명, 사용의 용이성, 융통성, 상호 운용성, 수출 가능성이다.

- S/MIME은 네트워크를 통해 주고받는 메시지에 대해 송수신자에게 전자우편 보안 서비스를 제공할 뿐만 아니라 MIME 객체를 전송할 수 있는 모든 프로토콜에서 보안 서비스를 이용할 수 있게 한다.

- 이러한 예로서 HTTP 프로토콜이 있다. HTTP 프로토콜은 MIME 객체를 전송할 수 있기 때문에 웹 보안을 위해 S/MIME을 사용할 수 있다.

```

보안 서비스 - 보안 메커니즘 - 암호 알고리즘

메시지 기밀성 - 암호화 - Triple-DES, RC2/40bit

메시지 무결성 - 해시함수 - SHA-1, MD5

사용자 인증 - 공개키 인증 - X.509 v3 인증서

부인방지 - 전자서명 - DSS, RSA

```

- S/MIME은 전자 서명을 위해 디지털 서명 표준(DSS)을 사용하고 세션키를 암호화하기 위해 Diffle-Hellman 알고리즘을 사용한다.

- 또한 전자서명과 세션키 모두를 암호화하는 알고리즘으로 RSA 알고리즘을 사용한다.

- 그리고 전자서명을 생성하기 위한 해시 함수로는 SHA-1과 MD5를 요구하고 Triple-DES 알고리즘이 메시지 암호화를 위해 사용된다.

- 공격자가 피해자의 브라우저에 스크립트를 실행하여 사용자 세션 탈취, 웹 사이트 변조, 악의적인 사이트로 이동 시킬 수 있다.

22. CSRF(Cross Site Request Forgery) - 로그온 된 피해자의 취약한 웹 애플리케이션에 피해자의 세션 쿠키와 기타 다른 인증 정보를 자동으로 포함하여 위조된 HTTP 요청을 강제로 보내도록 하는 것이다.

- 이것은 공격자가 취약한 어플리케이션이 피해자로부터 정당한 요청이라고 오해할 수 있는 요청들을 강제로 만들 수 있다.

23. Wep Proxy - 웹을 이용한 해킹 공격에 가장 활발히 사용되는 도구로, 클라이언트가 요청한 HTTP Request, Response 정보를 가로채어 필터링을 우회하거나 서버에 전송되는 데이터를 조작해

- 허가되지 않은 곳에 정보를 훔쳐보거나 변경, 삽입 등을 가능하게 한다.

24. robots.txt - 웹 사이트 검색엔진의 검색로봇이 홈 페이지에 접근(크롤링)하는 것을 방지하려 할 때 도메인 루트 디렉터리에 작성하는 파일명

25. 데이터 디들링(data diddling) - 프로그램에서 처리할 원시 데이터 자체를 위/변조하거나 미리 바꿔치기 할 데이터를 준비 해 두었다가 데이터를 추가하는 수법으로 

- 데이터의 검사, 변환, 처리 등 데이터에 접근이 가능한 모든 내부인에 의해 발생할 수 있다.

26. 은닉 채널(Convert Channel) - 기본 채널에 기생하여 다른 사람들에게는 보이지 않게 메시지를 주고 받는 행위를 말한다.

- 대표적으로 멀티미디어 데이터(이미지 등)에 메시지를 숨기는 스테가노그래피나 ICMP, DNS, TCP 등의 통신 프로토콜을 이용하여 메시지를 숨기는 방식이 있다.

27. 논리 폭탄(Logic Bomb) - 프로그램에 특정 날짜나 시간과 같은 조건을 설정하고 그 조건이 만족되면 악의적인 기능을 수행할 수 있도록 의도적으로 삽입된 코드를 말한다.

28. 집성 - 낮은 보안 등급의 정보를 조각별로 조합하여, 높은 등급의 정보를 알아내는 방식으로 개별 항목의 정보보다 종합적인 정보의 보안 등급이 높은 경우 심각한 문제가 발생한다.

29. 추론 - 접근 가능한 정당한 사용자 계정으로 수집한 정보를 통해 유추하여 높은 보안등급의 정보에 접근하는 방식

30. 접근 통제 - 인증 된 사용자에게 허가된 범위 내에서 데이터베이스 시스템 내부의 정보에 대한 접근을 허용하는 기술적 방법을 말한다.

31. 추론 통제 - 통계적인 자료에서 주로 발생하는 것으로, 간접적으로 노출된 데이터를 통해 다른 데이터가 공개되는 것을 제어하는 방법을 말한다.

32. 흐름 통제 - 정보의 흐름을 볼 때 보안등급이 높은 객체에서 낮은 객체로 정보가 흐르는 것을 제어한느 방법을 말한다.

33. skip-networking 옵션 - MYSQL 사용 시, 외부 네트워크에서 사용자의 접근을 제한하고 로컬에 설치된 어플리케이션에 의해서만 접근하도록 보안 설정 (my.cnf -> mysqld -> skip-networking)

34. PKI(Public Key Infrastructure) - 전자상거래 등의 E-Business 환경에서 공개키 암호화 기술을 지원하는 기반 구조

- Message Digest(해쉬함수, 메시지 축약)

- Symmetric Key Algorithm(대칭키)

- Asymmetric Key Algorithm(비대칭키 - RSA)

- 비대칭키를 사용할 때 통상적으로 공개키는 암호화할 때 사용, 개인키는 복호화 할 때 사용 

- 전자서명은 송신자의 개인키로 암호화, 공개키로 서명 검증 

35. WPKI(Wireless Public Key Infrastructure) - 무선 인터넷상에서의 인터넷 뱅킹, 사이버 주식 거래 시 외부 침입이나 정보 누출로부터 보호받을 수 있도록 하는 무선 인터넷 공개키 기반 구조

36. SET - 1997년 Visa와 Master카드사가 신용카드를 기반으로 한 인터넷 상의 전자 결제를 안전하게 이룰 수 있도록 마련한 전자 결제 과정 표준안으로 채택하였다.

- 전자 서명이나 해시 알고리즘을 이용하여 데이터의 무결성을 확보하고 전자서명, 인증서를 사용하여 카드 소지자 및 상점을 인증하여 서로 신뢰하고 거래를 할 수 있도록 도와준다.

- 이 때, SSL의 지불 정보가 상인에게 노출되는 문제점을 해결하기 위해 이중 서명을 사용한다.

- 고객은 상인에게 주문정보와 결제정보를 모두 전송한다.

- 그러나 상인은 결제정보를 확인할 수 없으며 변경할 수도 없다.

- 상인이 결제정보를 변경하지 않았고 고객이 보낸 데이터 원본임을 증명하기 위하여 이중서명을 하게 된다.

- 따라서 카드 매입사에서는 이중서명을 확인해 봄으로써 결제 정보의 무결성을 확인하게 된다.

37. 일반적으로 DNS는 UDP 53번 포트를 이용하여 서비스가 제공된다. 하지만 TCP 53번 포트를 이용할 경우가 있는데 이 경우는?

- 존 전송(Zone Transfer) - 마스터에 있는 원본 존 데이터를 슬레이브가 동기화/복사하는 과정을 존 전송이라 한다. 존 전송은 일반적으로 많은 존 데이터를 이용해야 하기 때문에 신뢰성 있는 전송을 보장하는 TCP를 이용한다.

- 응답 데이터가 512바이트를 초과하는 경우 - 클라이언트 질의에 대하여 네임서버 처리 결과 512 바이트를 초과하면 네임서버는 신뢰성 있는 전송을 보장해주기 위해 TCP로 재 요청하도록 응답한다.

- Set-Cookie - 서버에서 쿠키를 생성하여 웹 브라우저에 전달할 때 사용하는 HTTP 응답 헤더

- Cookie - 웹 브라우저가 쿠키를 서버로 전달할 때 사용하는 HTTP 요청 헤더

- httponly - 클라이언트에서 스크립트를 통한 해당 쿠키 접근을 차단하기 때문에 쿠키 탈취를 위한 XSS 공격에 효과적으로 대응할 수 있다.

- secure - 클라이언트에서 HTTPS(SSL/TLS) 통신일 경우에만 해당 쿠키를 전송하기 때문에 전송 중에 평문 쿠키가 노출되는 것을 방지할 수 있다.

38. dig axfr @192.168.19.13 kiwi99.com - 19.168.19.13 네임서버에게 kiwi99.com 관련 Zone 정보를 버전에 상관없이 무조건 요청한다.

- 대응방안 - Master 단독으로 운영할 경우에는 Zone Transfer를 허용하지 않도록 allow-transfer {none;} 설정하고, Slave가 구성되어 있다면 allow-transfer{slave_ip}; 설정한다.

# 네트워크 보안

1. 회선제어 - 점대점 또는 멀티 포인트 회선 구성방식과 단방향, 반이중 및 양방향 등의 통신방식에 따라 사용되는 전송링크에 대한 제어 규범이다.

2. 흐름제어 - 수신 장치의 용량 이상으로 데이터가 넘치지 않도록 송신장치를 제어하는 기술이다.(정지 - 대기, 슬라이딩 윈도우 기법)

3. 오류제어 - 여러가지 원인(전원, 주파수 혼란, 감쇠, 잡음 등)으로 인해 전송된 데이터가 발생할 수 있는 오류에 대한 해결을 위한 제어방식이다.

4. CSMA(Carrier Sense Multiple Access) - 동시에 네트워크를 사용하고자 할 때, 가장 중요한 것중에 하나가 상호 충돌을 방지하는 기술이다.

- 이를 위해서는 전송하기 전에 다른 호스트가 사용중인지를 선 체크해야 하는데 전송 선로에 흐르는 신호를 감지하여 해당 기능을 구현한다.

- 이러한 기능을 감지하는 프로토콜을 CSMA 프로토콜이라고 한다. (트래픽이 정상적으로 전송하기 위한 신호등같은 기능?)

- CD는 Detect, 충돌 감지 기능을 사용해 충돌 여부를 확인하는 것 - 이더넷 전송 프로토콜에서 사용

- CA는 Avoidance, 즉 충돌을 회피하는 기능을 사용하는 방식 - 무선 랜 전송 프로토콜에서 사용

- Backoff Time - 충돌 발생 후, 대기했다가 전송하기까지의 시간

5. 충돌 영역(Collision Domain) - 물리적 매체에 연결된 장치들의 집합, 2개의 장치가 동시에 매체에 접근하려고 하는 경우 두 신호가 충돌하는 것을 말한다.

6. Broadcast Domain - 다른 장치의 브로드캐스트 메시지를 수신하는 장비들의 집합

7. 포트 미러링 - 스위치 환경에서 모니터링과 트래픽 분석을 위해 특정 포트에 관한 패킷을 특정 포트로 전달하는 것

8. 허브 - 한 포트에서 수신한 신호를 Broadcast로 신호를 전파할 뿐. 기타 제어기능이 없고, Collision과 Broadcast Domain을 구분하지 못한다.

9. 스위치 - 전송방식을 이용하여 한 포트에서 수신한 신호를 Broadcast로 신호를 전파하지 않고, 특정 포트로만 신호를 전파한다.

- 확장 제한이 없으며, 이중화 시, 스패닝 트리 프로토콜을 이용하여 무한 루핑을 방지할 수 있다.

10. OSI Layer 2의 MAC 주소에서 브로드캐스트 주소는 FF:FF:FF:FF:FF:FF 이다.

11. 네트워크 계층 - OSI 계층 중 데이터가 목적지까지 올바르게 도달할 수 있도록 경로 선택 및 라우팅 기능을 수행하는 계층 

```

Layer 7 : 응용 계층(Application) / Data or Message

Layer 6 : 표현 계층(Presentation) / Data or Message

Layer 5 : 세션 계층(Session) / Data or Message

Layer 4 : 전송 계층(Transport) / Segment / L4 Switch

Layer 3 : 네트워크 계층(Network) / Packet / 라우터, L3 Switch

Layer 2 : 데이터링크 계층(Data Link) / Frame / L2 Switch(스위치), 브릿지

Layer 1 : 물리 계층(Physical) / Bit / 허브, 리피터

```

12. VLAN(Virtual LAN) - 데이터링크 계층에서 브로드캐스트 도메인을 나누기 위해 사용하는 기술

- 각 스위치들은 하나의 브로드캐스트 프레임에 대하여 동일 브로드캐스트 그룹이 아닌 곳에는 전달하지 않는다는 조건을 바탕으로 한다.

- 가상의 브로드캐스트 도메인을 생성함으로써 전체 브로드캐스트 트래픽을 제한할 수 있다.

```

Port 기반 VLAN : 스위치 포트를 각 VLAN에 할당하는 것으로, 같은 VLAN에 속한 포트에 연결된 호스트들 간에만 통신이 가능한 VLAN이다. 가장 일반적이고 많이 사용하는 구성이다.

MAC 기반 VLAN : 각 호스트들의 MAC 주소를 VLAN에 등록하여, 같은 VLAN에 속한 MAC 주소의 호스트들 간에만 통신이 가능하도록 구성한 VLAN이다.

IP주소 기반 VLAN : 네트워크 주소별로 VLAN을 구성하여 같은 네트워크에 속한 호스트들 간에만 통신이 가능하도록 구성한 VLAN이다.

프로토콜 기반 VLAN : 같은 통신 프로토콜(TCP/IP, IPX/SPX)을 가진 호스트들 간에만 통신이 가능하도록 구성한 VLAN이다.

```

13. IPv4에서 IPv6로 전환하는 기술

- 듀얼 스택(Dual Stack) - IPv4와 IPv6 프로토콜을 동시에 설정하여 통신 상대에 따라 선택적으로 사용할 수 있도록 하는 방식

- 터널링(Tunneling) - IPv4 네트워크를 경유하여 IPv6 네트워크 간 통신을 위한 방식으로 IPv4 네트워크를 통과하는 가상의 경로(터널)를 만들어 통신하는 방식

- 주소/헤더 변환 - 변환 헤더를 추가하여 IPv4 주소를 IPv6 주소로 변환하거나, IPv6 주소를 IPv4 주소로 변환하여 통신하는 방식

14. ICMP(Internet Control Message Protocol) - IP 프로토콜의 비 신뢰적인 특성을 보완하기 위한 프로토콜로 IP 패킷 중 에러 발생 시, 에러 발생 원인을 알려주거나 네트워크 상태를 진단 해 주는 기능을 제공한다.

15. IGMP(Internet Group Management Protocol) - IP 멀티캐스트 그룹에서 호스트 멤버를 관리하는 프로토콜

16. ICMP를 이용한 대표적인 Tool - ping, traceroute

- ping - ICMP 프로토콜의 Echo Request와 Echo Reply 메시지를 이용하여 호스트 활성화 여부를 검사해보고 RTT(Rount Trip Time)을 알아본다.

- traceroute - ICMP 프로토콜을 이용하여 TTL 값을 1로 설정하여 이 패킷을 받는 첫 번째 라우터에 의해 시간이 초과되도록 설계되어 있으며, 첫 번째 라우터는 Time Exceeded 메시지를 반환한다.

- 패킷이 목적지에 도달하게 되면 ICMP Destination Unreachable 메시지가 반환됨으로써, 마지막 홉에 걸린 시간을 측정할 수 있게 된다.

17. netstat - 침입 경로를 확인하기 위한 방법 중 현재 시스템과 통신을 하고 있는지 확인할 때 사용하는 명령어

- 네트워크 인터페이스에 대한 정보, 시스템의 라우팅 테이블 정보, 소켓 사용정보, 각 프로토콜 별 통계 정보 등 지정 옵션에 따라서 다양한 네트워크 정보를 제공한다.

18. 포트 스캐닝 - 임의의 호스트에서 어떤 포트를 사용하는지 원격으로 조사하는 공격

19. TCP FIN / NULL / Xmas - TCP 포트가 열린 경우에는 아무런 응답이 없고, 포트가 닫힌 경우에만 패킷이 되돌아 오는 원리를 이용한 스캔 방법 (SYN 스캔의 경우 닫힌 경우에 RST, 그러나 열린 경우에도 SYN + ACK 응답이 오기 때문에 해당 X)

20. 트리누(Trinoo) - UDP Flooding DDoS 공격을 유발하는데 사용한다.

21. P2P 봇넷 - 분산형 명령/제어방식, 참여 멤버(좀비/봇) 모두 C&C 역할을 수행하므로 별도의 도메인과 C&C 서버가 불필요하다. 대표적인 봇넷으로 Storm, Peacomm 등이 있다.

22. Slow HTTP {} DoS 공격

- Header - 다수의 HTTP 요청 시 요청 헤더의 끝을 의미하는 개행문자(CR,LF %0D%0A)를 전송하지 않고 불 필요한 헤더 정보를 천천히 지속적으로 전송하여 대상 웹 서버와의 연결 상태를 장시간 지속시킴으로써 연결 자원을 모두 소진시키는 형태의 서비스 거부 공격

- POST - 다수의 HTTP 요청 시 Content-Length 헤더를 매우 크게 조작한 후, 소량의 데이터를 천천히 지속적으로 전송하여 대상 웹 서버와의 연결 상태를 장시간 지속시킴으로써 연결 자원을 모두 소진시키는 형태의 서비스 거부 공격

- Read - 다수의 HTTP 요청에 대한 응답 시 조작된 "Zero Windows Packet"을 천천히 지속적으로 전송하여 대상 웹 서버와의 연결 상태를 장시간 지속시킴으로써 연결 자원을 모두 소진시키는 형태의 서비스 거부 공격


23. WEP(Wired Equivalent Privacy) - RC4, WPA(Wi-Fi Protected Access) - RC4, TKIP, WPA2 - AES, CCMP

24. VPN에서 터널링 기술 중 2계층인 데이터링크에서 사용하는 PPTP(Point-to-Point Tunneling Protocol) 기술은 MS사가 개발한 프로토콜로서 TCP 연결을 사용하며 IP, IPX 트래픽을 암호화하고 IP 헤더를 캡슐화하여 인터넷을 경유하여 전송하며, 하나의 터널에 하나의 연결만 지원한다.

26. VPN 기술 중 네트워크 계층에서 사용하는 대표적인 프로토콜로 IPSec가 있따.

- IPSec 프로토콜의 세부 프로토콜 중 AH 프로토콜은 송신처 인증, 무결성, 재전송 방지 등의 보안서비스를 제공하고

- ESP 프로토콜은 암호화를 통한 기밀성까지 제공한다.

27. IPSec은 네트워크 계층에서 다양한 보안 서비스를 제공해주는 프로토콜로, 두 가지 동작 모드를 가지고 있다.

- 전송 모드 - IP 헤더를 제외한 IP 패킷의 페이로드를 보호하는 모드

- 터널 모드 - IP 헤더를 포함한 IP 패킷 전체를 보호하는 모드

- IPSec의 키 교환은 IKE(Internet Key Exchange) 프로토콜을 사용한다.

29. SSL (Secure Socket Layer) - 넷스케이프사가 처음으로 만들었으며, 이 프로토콜은 신뢰할 수 있는 종단-대-종단의 안전한 서비스를 제공하기 위해 TCP를 이용하도록 설계하였으며, 인증 및 암호화 기능을 갖고 있다.

- 웹 브라우저와 웹 서버 간에 서로 상대의 신원을 확인하는 기능이 있다.

```

Handshake 프로토콜 : 양 종단간에 보안 파라미터를 협상하기 위한 프로토콜

Change Cipher Spec 프로토콜 : 협상된 보안 파라미터를 사용 가능하도록 적용/변경하기 위한 프로토콜

Alert 프로토콜 : 통신 과정에서 발생하는 오류를 통보하기 위한 프로토콜

Record 프로토콜 : 적용된 보안 파라미터를 이용하여 실제 단편화/압축/MAC/암호화 등을 수행하는 프로토콜

```

30. 스위치 재밍 - 스위치 장비의 MAC 주소 테이블을 오버플로우 시켜서 스위치가 허브처럼 동작하게 하여 패킷을 스니핑하는 기법

31. ARP Cache Poisoning - 공격자가 특정 호스트의 MAC 주소를 자신의 MAC 주소로 위조한 ARP Reply 패킷을 만들어 희생자에게 지속적으로 전송하면 ARP Cache Table에 특정 호스트의 MAC 주소가 공격자의 MAC 주소로 변조된다.

33. 스머프 공격 원리 - 출발지 IP를 희생자 IP로 위조하여 증폭 네트워크로 ICMP Echo Request를 브로드캐스트 함으로써 다수의 ICMP Echo Reply가 희생자에게 전달되어 서비스 거부를 유발시키는 공격이다.

- 방화벽이나 IDS / IPS 없이 막을 수 있는 방법 ? - 다른 네트워크로부터 자신의 네트워크로 들어오는 Directed Broadcast 패킷을 허용하지 않도록 라우터 설정을 한다. (no ip Directed Broadcast)

- 브로드캐스트 주소로 전송 된 ICMP Echo Request 메시지에 대해 응답하지 않도록 시스템 설정을 한다.

# 정보보안관리/법규

1. 기밀성 - 오직 인가된 사람, 프로세스, 시스템만이 알 필요성에 근거하여 시스템에 접근해야 한다는 성질
